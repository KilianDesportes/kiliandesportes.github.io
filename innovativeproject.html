Summary

During the whole semester, we had to work, in groups of students from different specialties, on an innovative project related to the Internet of Things. Our subject, named "Edge Computing Car Park", had as a goal to propose an application to easily find parking spaces on the INSA campus. To do this, we were asked to use cameras and image recognition to detect free parking spaces, then format and make this information accessible. We shared the work between each member of the group and I will present here the work I personally did.

Innovative Project - Edge Computing Car Park

The goal of our semester project was to propose a solution to find free parking spaces on campus. We used cameras filming the parking spaces and then applied an image recognition algorithm on these videos. This allows us to detect free parking spaces, then, using a server, we communicate this data to a mobile application, also developed by our team.
For this project, we worked in a team of 6, divided into two subteams of 3, one for the image acquisition and recognition part, the other on the application.
I was personally in the image acquisition and recognition team, in charge of implementing the free space detection algorithm and formatting the data for sending. On this page, I will develop a little more in depth all that relates to the organization of the project and also to the technical part that I realized.

Project Management and Organization

Concerning the project organization, we worked with the agile method throughout the project. This method allows us to have a close contact with the client to allow us to develop an application that suits him. So we had to plan weekly meetings to get feedback on what we were doing. Our client being also our technical tutor, these meetings also allowed us, especially at the beginning of the project, to have an informed opinion on the technologies we were going to use, as they were not imposed.

For the distribution of work within the group, we were asked to each work on a specific part of the project, in order to integrate our work at the end. This working method may not seem to be in line with the agile previously mentioned, but our project was better suited to this type of planning, given the different parts to be developed and the 'little' functionality we had to achieve. We separated the project into two main parts, image acquisition and recognition, and the mobile application. Two groups of 3 students working on each part. The more precise task distribution was then done within these two subgroups, and I personally had as a task the part on image recognition concerning whether the places were free or not, as well as the formatting of this information. My work was between Teo, in charge of detecting the parking places (my input) and Yanis, in charge of the communication and server part, which used my formatted information (my output).
Concerning my own organisation, I tried as much as possible to set a new goal each week to move forward, as the weekly meetings with the client went on. This organization was sometimes modified according to the work of others member of the team or other projects (related to other classes) that we had to carry out.

Technical Development

During this project, I had to work on the technical part related to the detection of free places or not. I had as input data the coordinates of the parking spaces in the image, in real time, work done by Teo. And I had to create a JSON file containing the different parking places and their status (free or not).

I based myself on a code produced by olgarose on GitHub: https://github.com/olgarose/ParkingLot .
This code, written in Python, allows you to detect parking spaces by hand from an image, then perform a detection of the parking spaces thanks to an image processing realized with the OpenCV library. The recognition is performed thanks to a series of functions that eliminate noise, smooth the image and then detect the curves inside the place selected later.
So I adapted this code to add several features.

At first, I had to change the way parking places were given as input, as manual detection was absolutely not satisfactory for automated use.
So I adapted the code inputs to allow me to give a file in which the coordinates of the different places are written, instead of giving an image for detection, which will replace the manual selection phase.

For a debug mode where the coordinates would be badly detected, I still kept the manual mode by modifying it so that instead of using an image as input, I can use the first image of the video stream provided. This allows to minimize the number of inputs on the code.

Once the input part was managed, I had to add different outputs, the code detecting parking spaces had no real output, as the output was a visual modification of the video (green parking places for the free places, blue is they are not). As this was not adapted to our use, I added a writing in a JSON file, a file that evolves in real time according to the current state of the parking spaces.

Once this code was done, with the necessary inputs and outputs, I had to move on to deploying it on the NVIDIA Jetson Nano card. This board needed a basic configuration, which I made, with the installation of an Operating System. For this I simply followed the different instructions on the NVIDIA website.
Once the board was ready, I had to configure and run the Raspberry Pi camera. I tested this camera thanks to different test codes provided by Raspberry Pi.
Once the tests were completed, I was able to integrate the camera into the code I had developed, by passing the live video stream as input instead of the mp4 videos I was used to working with. The integration went smoothly and the stream was well managed.
