Middleware and Service

During this training unit, we learned the concepts of cloud computing, web microservices (and REST API) and middleware for the Internet of Things.

We practiced cloud computing with OpenStack and Dockers tools to learn how to deploy and use cloud computing in an Internet of Things context.
We also learned the main standards for the Internet of Things, on both managing and communicating sides of connected objets. We practiced more deeply oneM2M standard to allow us to create a complete architecture for Internet of Things by representing sensors and their datas and by allowing them to be interconnected.
We also learned how to setup and use microservices to interconnect these objects by using REST API.

The aim of taking all these tools in hand is to enable us to set up a complete installation of action automation based on the Internet of Things. This was the final topic of the training unit, with the automation (lights, windows, heating,...) of a classroom with different virtual sensors.

Middleware and Service Traning Unit

Adaptability: Cloud and Autonomic Computing

This material was not carried out via one project but through several practical works covering a range of cloud computing technologies. We were able to discover and practice cloud computing with OpenStack, a private cloud service, and a Docker, a containerisation service.
During these practical works, we were able to take charge of the operation and implementation of a cloud for the deployment of applications (with microservices), the SaaS part of cloud computing, as well as the implementation of the infrastructure (VM, containers), the IaaS part of cloud computing.
We have also seen how to organise the different VMs in a more complex network than a simple local network, in order to better manage the different services.
All these experiences will have given us a good understanding of the stakes and possible uses of cloud computing.

Middleware for the Internet of Things

During this course, we went through 4 tools used in internet of things : MQTT Protocol, Sierra Wireless MangOH, oneM2M and Node-Red.
We initially took in hand MQTT and its mechanism of publish/subscribe to then set it up with an arduino to be able to light in LED via a broker and topic MQTT.
In a second time, we were able to take in hand the mangOH cards of Sierra Wireless. This allowed us to see that there were microcontrollers that allowed us to perform many functions simply, all with access to a graphical interface with simple parameterization (creation of functions, activation of sensors, ...etc) but also dashboard-type graphical interfaces, with measurement curves for example.
We then discovered and used the oneM2M standard which allows, through the use of REST queries, to create a very useful resource tree for the internet of objects, which allows to store both objects but also the data linked to it, via different entity levels. This standard will allow to centralize the information related to a network of connected objects.
Finally, we used Node-Red and nodejs to create a gateway between the oneM2M standard and MQTT, with the creation of data entities to light lamps following different MQTT requests arriving on several topics. We can also note that node-red could have been used to create a dashboard to represent these different functions.
We have therefore been able, through these different works, to take in hand many tools to represent and use networks of connected objects at the application level.

Service Oriented Architecture

This course allowed us to learn how to develop and implement a service based architecture. We learned how to implement web services, first based on a deployment on a tomcat server and using the REST API. We then used the SpringBoot framework to automate and simplify the development and deployment of these web services using the REST API.
We have also learned how to interconnect these different services to enable the use of services in other services.

In another part of the course, we learned how to implement continuous integration in our projects with the use of Jenkins. Through Jenkins, we were able to set up unit test automation, the use of Git and finally the deployment of different applications. We also learned how to set up pipelines to be able to carry out all these automation in a harmonized way.

We were also introduced to Jira for project management. This tool will be used and further developed in the training unit project.

Project : Automation of the management of connected objects in a room

During this project, we will set up an IoT platform to automate the management of a classroom. The automation will be done on functions such as lighting, heating and door locking.
We will use the oneM2M standard to simulate different sensors present in the classroom.
For the use of these sensors and their management, we will use web services with SpringBoot to retrieve and process the various data emitted by its simulated sensors to be able to perform actions based on threshold functions.
We will, throughout the project, use Jira to allow a project management organized on the Scrum method, in agile, with spring of 2 weeks.

Developped skills and knowledge

Adaptability: Cloud and Autonomic Computing


  • Understand the concept of cloud computing

During the practical work, we were able to carry out a theoretical analysis of what cloud computing and its various components (SaaS, IaaS, PaaS) were, as well as its link with virtual machines and container services.

  • Use a IaaS-type cloud service

We were able to take over and use the OpenStack platform, which allows us to create cloud environments (VM instances) in a simple way via an overlay on the server infrastructure (IaaS).

  • Deploy and adapt a cloud-based platform for IoT

During tp, with openstack, we were able to set up a complete cloud environment, with an architecture containing several networks, to host and use several microservices that interacted with each other and with a user.

Middleware for the Internet of Things


  • Know how to situate the main standards for the Internet of Things

During this course, we were able to theoretically discover several standards both for the management and representation of connected objects or the communication between them. During the oneM2M MOOC we discovered several standards in a theoretical way. We were then able to use MQTT and oneM2M in more detail during the praticals works and the project.

  • Deploy an architecture compliant to an IoT standard and implement a sensor network

We were able, both during a practical work and also during the project, to deploy a connected object architecture using the oneM2M standard and the REST API. Then, especially during the project, we set up services so that these objects could communicate and interact with each other.

  • Deploy and configure and IoT architecture using OM2M

During the project, we were able to implement a simulated IoT sensor architecture using the oneM2M standard. These sensors were arranged in different rooms and had containers for the different data they sent. These different data were represented by content instances. This allowed a clear, precise and functional representation of the simulated sensor network that we wanted to implement.

  • Interact with the different resources of the architecture using REST services

To communicate with the different connected objects of the architecture, we had to use REST services to access the different content instances of the sensors (the last data collected by this sensor) and allow to perform different actions. We also used the REST API to implement this sensor architecture.

  • Integrate a new technology into the deployed architecture

During the last practical work, we were able to make the two standards MQTT and oneM2M interact with node-red to allow actions in the oneM2M architecture on data reception in some MQTT topics. However, we did not integrate new technologies in the project, we only used oneM2M with REST services.
