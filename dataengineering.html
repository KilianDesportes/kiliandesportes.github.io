Summary

During this training unit, we learnt how to handle different types of data. Firstly, by learning the semantic web, we learnt how to accurately characterize data in order to be able to classify it and make it more usable. To do this, firstly, we learnt how to define an ontology and use it with a reasoner to infer information. We also see how to create and improve an ontology with Java (Apache Jena). Finally, we have seen how 'large' ontology are designed, looking at SSN ontology, an ontology for sensors.

Secondly, we learnt how to manipulate large databases thanks to the R language, a big data manipulation language. We learnt how to extract from these large databases the information we wanted in order to be able to format them and make the easily understandable with different types of plot.

We have applied these learnt skills during a big data project, where we were free, from a chosen data set, to extract what we jugged pertinent to show.

Analysis and data processing, business applications

Processing Semantic Data

We worked on the semantic web, initially using Protégé, a tool for creating and manipulating ontologies. We had to create an ontology related to the weather and its observation, then use the Protégé reasoner to infer knowledge about this ontology.

In a second step, we used Apache Jena to start from an existing dataset (sensor readings) and enrich these data by creating links between them. We then exported the knowledge we had established in our ontology, and observed the result in Protégé.

Data Processing and Analysis : Big Data

During the practical work, we first learned the R language. We first learned how to create and manipulate vectors, then functions and finally use a data set. We learned how to use data sets, extract information from them and then display it. We mainly used the ddply function of the plyr package to manipulate data sets, and ggplot to display our results in graphical form.

Once our introduction to R has been completed and the important tools assimilated, we were asked, during a project, to find a data set on the internet and manipulate it to extract the information we considered relevant.
For this project, we choosed to analyze a data set for video games sales. We had access to sales per "major regions" (NA,EU,JP,Others) as well as platform, editor or game genre. The project report is available below :

Developped skills and knowledge

Software Engineering


This skill is more related to Service oriented architecture and Innovative training units.

  • Define the different phases in software development

During projects we realised this year, like the innovative project or the room automation, we had to develop our knowledge of the software lifecycle to enable us to successfully complete our project, starting with the definition of the specifications and the design of the software, followed by the implementation and tests to end the cycle with the deployment. To help us to automate these different phases, we learnt to use Jenkis, a CI/CD tool.

  • Know the different project management methods

During this year, we had practice Agile and Scrum project management methods. We also learnt other methods of project management like waterfall or different methods related to task scheduling (critical path method for example).

  • Apply one of these methods a project

For the innovative project, we worked with agile methods to enable us to develop a project in accordance with the client's request, thanks to weekly meetings. During the room automation project, we have implemented the Scrum method. We used Jira to create our user stories and specifications and organise them in different sprints, sprints that we were able to carry out during the different practical sessions.

Processing Semantic Data


  • Design and understand a model for an application

During the practical work, we had to design an ontology for the weather and its observations. We then used it to infer knowledge from the reasoner.

  • Be able to design data acquisition system with respect to the application.

During the Service Oriented Architecture project, we had to create the whole data acquisition system for our room management. We simulate everything using oneM2M, both sensors and actuators. I marked 1 'check' as red because we did not have to respect the application, as we were also designing it, we had no constraint on our data acquisition system.

  • Know how to infer new knowlegde from a knowledge base

During the first practical work, using Protégé, we used the reasoner Hermit to infer knowledge from our ontology. Some example are available in the practical work report presented above.

  • Be able to enrich data with semantic meta-data

During the practical work, we had to use an existing data set produced by a city. First we convert this 3-stared CSV data set into a 5-star with links between different measures. To do this we used Apache Jena to be able to construct relations between measures, using ontology related object and data properties. Then with these new relations, we enrich the dataset and export it to exploit it in Protégé.

Data Processing and Analysis: Big Data


  • Know how to explore and represent data sets

During the practical works, we used multiple CSV data set and we learnt how to explore and manipulate them using ddply function (from plyr package). We also learnt to represent the data using ggplot, mostly for bar and line charts.

  • Master R

We worked, during our practical work and our project, with the R language, to learn how to process data and make them easily interpretable with plotting. With all these works, we learn the main principles and functioning of this language.

  • Master complexity associated to statistical data processing and know the techniques to be used to minimise them.

During lectures, we saw how what is the problem related to complexity in data processing. We briefly learned how to compare problems with asymptotic complexity and how to check what is taking time during code execution. However, we did not use it during our work, that is why I don't really consider this skill as fully acquired.
